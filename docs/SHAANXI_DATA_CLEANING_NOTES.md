# 陕西省教学成果奖数据整理问题记录

本文档记录 `陕西省教学成果奖_295项_总表.xlsx` 在整理过程中出现过的典型错误、原因和修正方法，供后续处理其他省份时复用。

## 一、已出现过的错误

### 1. 长成果名称被 PDF 自动换行，后半段被误拆

表现：
- 成果名称只剩后半截，前半截丢失或断开
- 成果名称后面混入完成人姓名
- 下一列的完成人出现缺漏

已出现的典型例子：
- 第 33 行：`以乡村振兴为命题，深化艺术社会实践课程体系在协同育人机制中的创新实践`
- 第 193 行：`守护华夏祖脉当好秦岭卫士：应用型高校大中小学思政教育共同体建设创新实践`
- 第 194 行：`资源重构院园共生数据循证：学前教育教学法类课程教学体系创新与实践`

原因：
- PDF 中文本换行后，抽取脚本把成果名称第二行误识别成完成人或下一字段
- 成果名称不会中断，但 PDF 中视觉上会分成两行，解析时必须重新拼接

修正方法：
- 遇到“序号 + 单位”后，连续的成果名称行必须先完整拼接，再识别完成人
- 对于明显连续的标题片段，不允许直接截断
- 如果标题后面混入姓名，要把姓名重新拆回完成人列

后续处理原则：
- 成果名称只要是同一条记录的连续文本，就必须视为一个完整标题
- 不允许因为换行把标题拆成两条字段

### 1.1 单位、标题、完成人在同一条原始文本里被整列错位

表现：
- `成果名称` 只剩很短一截，甚至直接变成单位名的一部分
- `完成单位` 里混入了完整标题
- `完成人` 或 `完成单位` 被整体右移

本次新增发现的典型例子：
- 第 62 行
- 第 200 行
- 第 220 行
- 第 222 行
- 第 253 行

原因：
- 有些 PDF 记录不是“单位一行、标题一行、作者一行”的固定结构
- 有些记录会出现：
  - `序号 + 单位 + 标题 + 作者` 全在一行
  - 单位跨多行
  - 标题跨多行
  - 作者跨多行
- 如果解析逻辑只按固定行数切，就会出现整列右移

修正方法：
- 不能假设每条记录都有固定行数
- 需要先识别“序号”起点，再动态判断：
  - 哪些行属于单位
  - 哪些行属于标题
  - 哪些行属于完成人
- 对已经生成的总表，要额外扫描：
  - `完成单位` 中是否出现 `培养 / 实践 / 体系 / 模式 / 构建 / 创新`
  - `成果名称` 是否异常短但 `完成单位` 异常长

后续处理原则：
- 对 PDF 转 Excel，不能只靠一次粗解析结果直接交付
- 必须再做一轮“错位扫描”

### 2. 完成人名单被截断或拆裂

表现：
- 完成人只剩后半段
- 完成人以单字或残片开头，例如 `丰、张乐...`
- 两个姓名黏连在一起，例如 `任一飞杨晨`

已出现的典型例子：
- 第 33 行：
  - 错误：`丰、张乐、吴彦臻、张亚谦`
  - 正确：`武小川、宋群、崔凯敏、任一飞、杨晨、郭庆丰、张乐、吴彦臻、张亚谦`

原因：
- PDF 抽取后，完成人有时分成两行
- 两行拼接时如果没有保留正确分隔符，就会出现“前半段丢失”或“姓名黏连”

修正方法：
- 完成人行需要连续拼接
- 拼接后要检查是否存在明显黏连姓名
- 如果第二个字符就是 `、` 或 `，`，通常说明这一行是残片，需回溯上一列修正

后续处理原则：
- 完成人字段不能以残片开头
- 完成人字段必须是完整姓名列表，不能出现标题尾部混入

### 3. 完成单位错位

表现：
- 完成单位为空
- 完成单位被前一列或后一列挤占
- 由于标题和完成人错位，完成单位跟着出错

已出现的典型例子：
- 第 193 行之前曾出现完成单位错位，后修正为 `商洛学院`

原因：
- 标题与完成人解析一旦错位，完成单位通常会连带错位

修正方法：
- 先纠正成果名称和完成人，再校验完成单位
- 完成单位应与原 PDF 中对应序号逐条核对

后续处理原则：
- 不单独修单位，要和标题、完成人一起联动检查

### 4. 基础字段空值

表现：
- 某些行只有成果名称，其他列几乎全空
- 或某条只有单位，没有完整标题/完成人

已出现的典型例子：
- 用户指出过第 95 行、第 98 行存在严重缺漏

原因：
- PDF 解析时把连续多行拆散
- 或某条记录被分割到多条伪记录中

修正方法：
- 以原 PDF 为准，按记录整体重组
- 修正后重新编号，并复检 `序号 / 成果名称 / 完成人 / 完成单位`

后续处理原则：
- 核心四列不允许为空

### 4.1 终端内联修复导致中文编码被写坏

表现：
- 通过终端临时脚本直接写 Excel 时，中文被写成 `????`

原因：
- 终端 here-string / 内联脚本在当前编码环境下不稳定
- 直接把中文常量塞进临时命令，有可能被控制台编码损坏

修正方法：
- 不再用终端临时内联中文字符串修 Excel
- 改为：
  - 在工作区保存 UTF-8 脚本
  - 或使用 ASCII 的 `\\uXXXX` 转义
  - 再由脚本写回 Excel

后续处理原则：
- 任何批量中文修正都必须通过文件脚本执行
- 不再直接用临时内联命令写中文到 Excel

### 5. 本科与研究生合并后顺序与编号问题

表现：
- 合并后顺序混乱
- 编号不连续

修正要求：
- 先本科，后研究生
- 统一连续编号 `1..N`

本次陕西最终处理结果：
- 本科 `215` 项
- 研究生 `80` 项
- 合计 `295` 项

## 二、陕西最终采用的字段结构

陕西最终总表保留以下列：
- `序号`
- `成果名称`
- `完成人`
- `完成单位`
- `第一完成单位层次`
- `第一完成单位学科类型`
- `第一完成单位地理位置`
- `获奖等级`

其中：
- `第一完成单位层次` 取完成单位第一项，分类为 `985 / 211 / 其他`
- `第一完成单位学科类型` 分类为：
  - `综合类`
  - `理工类`
  - `师范类`
  - `医药类`
  - `财经类`
  - `农林类`
  - `艺术类`
  - `政法类`
  - `文法类`
  - `语言类`
  - `体育类`
  - `其他类`
- `第一完成单位地理位置` 取完成单位第一项所属城市

## 三、以后处理其他省份时的检查清单

每次提取完原始总表后，至少检查以下内容：

1. 核对总条数是否和公告一致
- 不只看文件行数，要看有效记录数

2. 抽查长标题
- 尤其是跨行标题
- 检查是否把标题后半段拆进了完成人

3. 抽查完成人
- 检查是否以残片开头
- 检查是否出现两个姓名黏连

4. 抽查完成单位
- 检查是否与对应成果名称匹配

5. 检查核心四列
- `序号`
- `成果名称`
- `完成人`
- `完成单位`
- 这四列必须无空值

6. 合并多个类别时检查顺序
- 如有本科/研究生/高职分类，先明确合并顺序
- 合并后重新连续编号

7. 再做一轮针对性扫描
- 扫描标题中是否混入大量姓名分隔符
- 扫描完成人是否以单字残片开头
- 扫描是否存在明显错位行
- 扫描 `完成单位` 中是否混入标题关键词
- 扫描 `成果名称` 是否异常短但 `完成单位` 异常长

## 四、建议的后续处理方式

以后处理新省份时，建议固定采用下面流程：

1. 从 PDF/Excel 抽出原始总表
2. 先只校正结构问题：
- 标题
- 完成人
- 完成单位
- 获奖等级
3. 再补扩展字段：
- 第一完成单位层次
- 第一完成单位学科类型
- 第一完成单位地理位置
4. 最后统一复检：
- 条数
- 编号
- 核心四列空值
- 长标题错位
- 完成人残片
